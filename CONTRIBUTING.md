# Contributing to RAG Vector DB Lab

Thank you for your interest in contributing! This is a learning and experimentation project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/rag_vectordb.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes with `docker-compose up --build`
6. Commit: `git commit -m "Description of changes"`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Guidelines

- Follow existing code style (TypeScript/Angular/NestJS conventions)
- Test your changes locally before submitting
- Update documentation if adding new features
- Keep commits focused and atomic

## Project Structure

- `backend/` - NestJS backend API
- `frontend/` - Angular standalone application
- `docker-compose.yml` - Service orchestration

## Running Tests

```bash
# Backend tests
cd backend
npm test

# Frontend tests
cd frontend
npm test
```
